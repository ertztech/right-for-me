# Work Selection

Purpose:
Choose the single highest-value next piece of work without expanding the framework or overloading Codex.

Use:
- Start from product direction.
- Check the current product state and the clearest user problem.
- Prefer work with strong value, manageable risk, and clear Codex readiness.
- Avoid work that is still vague, coupled, or likely to create avoidable rework.

Quick filter:
1. Does this help the user make better job-fit or application decisions?
2. Is it the highest-value next improvement in the current product state?
3. Is the risk acceptable for one focused implementation pass?
4. Is the work clear enough to hand to Codex without major guessing?
5. If not, what is the smaller, more ready version?

Output:
- Recommended next work item
- Why now
- Main risk
- Codex readiness
- What to defer
