# Workflow Examples

Workflow files remain authoritative.

These examples demonstrate preferred structure, density, role interaction, and decision boundaries for `.ertztech` workflows. Adapt them to the situation instead of copying them mechanically.

Guidelines:

- Preserve meaningful disagreement when it materially affects the result.
- Do not manufacture debate just to make the session feel rigorous.
- Run the full workflow without narrating unnecessary ceremony.
- Use normal inline Markdown.
- Use individual code blocks only for reusable artifacts.
- Distinguish proposed, approved, and executed work clearly.
- Treat newly authored examples as reference examples, not historical transcripts.
