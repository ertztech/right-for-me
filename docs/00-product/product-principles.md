# Product Principles

- Simplicity over feature count.
- Enter information once and reuse it everywhere.
- Honest confidence beats inflated confidence.
- Every screen should make the next action obvious.
- Opportunities are the center of the workflow.
- Motion should reinforce progress, not decorate the interface.
- AI should remove duplicate work and uncover evidence the user already has.
- The dashboard should act as a launchpad, not a static report.
- Returning users should move fast without distraction.
- Features should support job fit, stretch, and application positioning before anything adjacent.
