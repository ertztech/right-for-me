## Pre-Implementation Review

### Product Owner Review
- [ ] Product decision is clear
- [ ] User value is clear
- [ ] MVP scope is small enough
- [ ] Nice-to-haves are out of scope or parked

### Architect Review
- [ ] Technical direction is clear
- [ ] Codex is not being asked to make architecture decisions
- [ ] No unnecessary coupling is introduced
- [ ] ADR need has been considered

### Engineering Manager Review
- [ ] Work is small enough for one Codex session
- [ ] Handoff context is complete
- [ ] Acceptance criteria are clear
- [ ] Out-of-scope items are explicit

### Software Engineer Review
- [ ] Implementation approach fits existing code patterns
- [ ] File/module boundaries are reasonable
- [ ] Testability has been considered
- [ ] No unnecessary code complexity is being introduced

### QA Review
- [ ] Acceptance criteria are testable
- [ ] Test checklist is clear
- [ ] Regression risks are named
- [ ] Done can be objectively verified

### Delivery Coach Review
- [ ] Workflow has been followed
- [ ] Documentation updates are identified
- [ ] Backlog updates are identified
- [ ] This is truly implementation-ready

### Founder Approval
- [ ] Final scope approved

## GitHub Issue Draft

## Codex Prompt

## Review Checklist
- [ ] Scope is implementation-ready
- [ ] Founder decisions are explicit
- [ ] Documentation and backlog updates are named
- [ ] Architecture impact is captured
- [ ] Acceptance criteria are testable
- [ ] Pre-Implementation Review is complete
- [ ] Codex can work without relying on chat history
