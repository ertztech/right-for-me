# Workflow Improvements

This page collects the current process adjustments that shape the repo-first operating system. Track future changes in [../04-backlog/process-improvements.md](../04-backlog/process-improvements.md) until they are adopted.

## Current Improvements

- Add a Documentation and Traceability checkpoint to implementation planning.
- Require an Implementation Package before creating a GitHub Issue.
- Keep repo docs as the source of truth for planning, backlog, architecture, process, and decisions.
- Limit GitHub Issues to one or two near-term implementation-ready tasks.
- Do not use GitHub Projects for now.
- Add a facilitator guide for planning and review sessions.
- Track workflow and process improvements in the repo backlog.
- Keep `.ertztech/workflows` operational and cross-link it from human-readable docs.
