# NextMove Docs

This folder is the repo-first source of truth for product planning, backlog, architecture, process, and key decisions. GitHub Issues are reserved for one or two near-term tasks that are already implementation-ready.

## Start Here

- Start with [Product Direction](00-product/README.md) for mission, principles, and roadmap context.
- Review current epics in [Backlog Epics](04-backlog/epics.md).
- Read [Product Operating System](03-process/product-operating-system.md) for how the team works.
- Use [Facilitator Guide](03-process/facilitator-guide.md) when running planning or review sessions.
- Draft handoffs with [Implementation Package Template](03-process/implementation-package-template.md).
- Run operational workflows from [.ertztech/workflows](../.ertztech/workflows).

## Documentation Map

- [00 Product](00-product/README.md): mission, principles, shared language, and direction.
- [01 Features](01-features/README.md): lightweight feature specs and experience decisions.
- [02 Architecture](02-architecture/README.md): system shape, ADR index, and data model references.
- [03 Process](03-process/README.md): the human-readable operating system and facilitator guidance.
- [04 Backlog](04-backlog/README.md): repo-managed epics, features, process improvements, debt, and parked ideas.

## Operating Model

Planning lives in this repo. The normal path is:

Idea -> Repo Backlog -> Implementation Planning -> Implementation Package -> GitHub Issue -> Codex -> Implementation Review -> Merge -> Docs and Backlog Updated
