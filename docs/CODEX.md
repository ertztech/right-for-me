# Codex Instructions

Before implementing new features, read:

- docs/PRODUCT_VISION.md
- docs/PRODUCT_DECISIONS.md
- docs/ROADMAP.md

## Product Rules

- Prefer simple UX.
- Avoid duplicate user input.
- Opportunities are the center of the workflow.
- Profile is the source of truth.
- Dashboard should guide the next action.
- Application materials should be generated from saved Profile and Opportunity data.
- Do not redesign the app unless requested.
- Keep future ideas in Roadmap / Not Yet instead of building them early.

## Development Rules

- Keep changes focused to the requested feature.
- Preserve existing tests.
- Add or update tests when practical.
- Keep AI Test Mode working.
- Do not expose API keys or secrets.
- Summarize files changed, tests run, and known limitations when finished.
