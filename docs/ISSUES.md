# Initial Issues

These are the first issues to create when the GitHub repository is live.

## Issue 1: Build v0.1 Job Fit and Stretch Analyzer

Create the first end-to-end user workflow:

- User can paste a job description.
- User can optionally paste background or resume text.
- App returns a fit score, stretch score, evidence, gaps, strategy, and recommendation.

Acceptance criteria:

- The workflow works without a backend.
- Results are readable and structured.
- The recommendation is honest, not blindly encouraging.

## Issue 2: Write Public README

Create a README that explains the mission, MVP scope, roadmap, and product principles.

Acceptance criteria:

- The README explains why RightForMe exists.
- The README describes v0.1 clearly.
- The README includes the product principle: does this help someone choose the right opportunity and get an interview?

## Issue 3: Add Roadmap

Document the planned path from v0.1 to public launch.

Acceptance criteria:

- Roadmap includes v0.1 through v1.0.
- Each milestone has a clear user-facing outcome.

## Issue 4: Add Application Strategy Output

Add a section that tells the user how to position themselves for the role.

Acceptance criteria:

- Strategy references job requirements.
- Strategy references user background when provided.
- Strategy does not invent experience.

## Issue 5: Add Saved Analysis

Allow users to save job analyses locally.

Acceptance criteria:

- User can save an analysis.
- User can view saved analyses.
- User can delete saved analyses.
